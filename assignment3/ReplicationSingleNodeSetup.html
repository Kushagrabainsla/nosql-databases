Deploying a MongoDB Replica Set of 3 members on a Single Server

Instructor: Dr. Kim

  • Reference: The Definitive Guide to MongoDB, 3rd Edition, by David Howes, et al.
  • A replica set to configure 
    Service	Daemon	Address	Dbpath
Member 1	mongod	[hostname]:27021	/db/active1/data
Member 2	mongod	[hostname]:27022	/db/active2/data
Member 3	mongod	[hostname]:27023	/db/passive1/data
  • Preperation: Create a Ubuntu Docker container dedicated to this deployment and setup MongoDB in the container. In the following instruction, $ is a prompt from the Ubuntu Docker container.
1. In the Docker container, create the necessary data directories for each member 
$ mkdir -p /db/active1/data /db/active2/data /db/passive1/data
2. Open three windows. Enter the Docker container in each window where you will start mongod working on the assigned port as shown below. 
$ sudo mongod --port 27021 --dbpath /db/active1/data --replSet testset
$ sudo mongod --port 27022 --dbpath /db/active2/data --replSet testset
$ sudo mongod --port 27023 --dbpath /db/passive1/data --replSet testset
